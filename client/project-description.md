### **Описание MVP (Minimum Viable Product) для Rentor**

**Цель MVP:** Базовый веб-сайт для аренды долгосрочного жилья, где арендодатели могут публиковать объявления, а арендаторы — находить их и связываться с арендодателями.

---

### **1. Ключевые Пользовательские Роли и Возможности**

#### **Для Всех Пользователей (Арендатор/Арендодатель)**
*   **Регистрация и Вход (REQ-UR):**
    *   Вход по email или номеру телефона **без пароля**.
    *   Система отправляет одноразовый код подтверждения (OTP) для входа. В MVP OTP отправляется **только по email**.
    *   Таймер на 60 секунд для повторной отправки кода.
    *   Обработка ошибок при неверном коде.
    *   Кнопка "Выйти" в интерфейсе.

#### **Для Арендодателей**
*   **Управление Объявлениями (REQ-MA):**
    *   **Создание:** Форма для заполнения данных об объекте (согласно модели данных `Advertisement`, включая фото, цену, описание, адрес, тип жилья и т.д.).
    *   **Просмотр:** Страница "Мои объявления" с списком (гридом) карточек своих объявлений.
    *   **Редактирование:** Возможность изменить данные своего объявления через форму.
    *   **Удаление:** Удаление объявления с подтверждением действия.
    *   Валидация всех полей формы на фронтенде и бэкенде.

#### **Для Арендаторов**
*   **Просмотр Каталога Объявлений (REQ-AV):**
    *   Главная страница или раздел "Объявления" отображает объявления в виде сетки карточек.
    *   Карточка содержит: **фото, цену, краткое описание**.
    *   Клик по карточке открывает страницу с подробной информацией об объявлении.
    *   Поддержка пагинации (постраничной загрузки).

*   **Страница Детализации Объявления (REQ-AC):**
    *   Отображение всей информации об объекте: галерея фото, полное описание, цена, адрес, характеристики (площадь, кол-во комнат).
    *   Кнопка **"Связаться с арендодателем"**. В MVP эта кнопка **не открывает чат**, а показывает контактные данные арендодателя (email/телефон) или инициирует простое уведомление.

*   **Поиск и Фильтрация (REQ-SF):**
    *   Интерфейс с формой фильтров на странице объявлений.
    *   **Базовые фильтры:** по цене (диапазон), типу жилья (апартаменты, дом, комната), ключевым словам.
    *   Возможность комбинировать фильтры (возможно, с некоторыми ограничениями в MVP).
    *   Сообщение "Объявления не найдены", если по запросу ничего нет.

### **2. Фронтенд Компоненты и Логика (UI/UX)**

1.  **Компонент Аутентификации:**
    *   Поле ввода: `email` или `phone_number`.
    *   Кнопка "Получить код". После нажатия — блокировка на 60 сек.
    *   Второй шаг: поле для ввода 6-значного `otp_code`.
    *   Валидация формата email/телефона. Обработка ошибок (неверный код, 401).

2.  **Компонент Профиля Пользователя (REQ-PD):**
    *   Форма для просмотра и редактирования личных данных (`User_Profile`: имя, фамилия и т.д.).
    *   Валидация полей согласно модели данных. Сообщения об ошибках (400 от сервера).

3.  **Компонент Каталога Объявлений:**
    *   Сетка карточек. Ленивая загрузка изображений.
    *   Панель фильтров и, возможно, сортировки.
    *   Пагинация ("Загрузить еще" или номера страниц).

4.  **Компонент Карточки Объявления (в списке):**
    *   Элементы: первое изображение из `Advantages`, `price`, `title` (или начало `description`).

5.  **Страница Детализации Объявления:**
    *   Галерея изображений.
    *   Отображение всех полей: `title`, `description`, `price`, `address`, `city`, `square`, `rooms` и т.д.
    *   Кнопка **"Связаться с арендодателем"** (логика описана выше).

6.  **Компоненты Арендодателя ("Мои объявления"):**
    *   Страница со списком его объявлений.
    *   Кнопка "Добавить объявление" → открывает большую форму.
    *   Форма создания/редактирования объявления со всеми необходимыми полями (включая загрузку фото).
    *   Кнопки "Редактировать" и "Удалить" на каждой карточке в списке.

### **3. Важные Технические Ограничения и Контекст для Фронтенда**

*   **Тип приложения:** Веб-сайт (не мобильное native-приложение).
*   **Поддерживаемые браузеры:** Chrome >= 87 / Firefox >= 78 / Safari >= 14 / Edge >= 88. Обязателен **адаптивный дизайн** для мобильных устройств.
*   **Стек:** Фронтенд на **TypeScript**.
*   **Взаимодействие с Бэкендом:** Через **REST API** (JSON over HTTPS). Все критические операции (создание, редактирование) возвращают коды ошибок (400, 401 и т.д.), которые нужно обрабатывать.
*   **Локализация для Казахстана (MVP):**
    *   Валюта: **KZT** (тенге).
    *   Формат даты: **DD.MM.YYYY**.
*   **Чат НЕ входит в MVP.** Вместо чата — отображение контактов или простой ответ по email.
*   **Платежи, рейтинги, рекламные баннеры, онлайн-подписание договоров — НЕ входят в MVP.**
